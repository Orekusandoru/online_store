Product Backlog: Online Store

1. Завдання для Продукту (Product Backlog)
  12 пріоритезованих завдань, необхідних для реалізації функціоналу згідно з технічним завданням:
  
  Налаштування середовища: Налаштувати середовище розробки, систему контролю версій (Git) та базову інфраструктуру (DevOps).
  
  Проєктування та створення БД: Спроектувати та створити реляційну базу даних  для зберігання товарів, категорій, клієнтів та замовлень.
  
  Реалізація Адмін-панелі (Категорії): Створити CRUD (Create, Read, Update, Delete) функціонал для управління категоріями.
  
  Реалізація Адмін-панелі (Товари): Створити CRUD функціонал для управління товарами (назва, опис, ціна, фото, характеристики).
  
  Публічний каталог: Реалізувати відображення каталогу товарів з розбиттям на сторінки (пагінація).
  
  Пошук та Фільтрація: Реалізувати функціонал пошуку за назвою та фільтрації товарів (за категорією, ціною).
  
  Функціонал Кошика: Реалізувати додавання товарів у кошик, зміну їх кількості та видалення.
  
  Оформлення замовлення: Створити процес оформлення замовлення (введення даних клієнта, вибір способу доставки та оплати).
  
  Реалізація Адмін-панелі (Замовлення): Створити інтерфейс для перегляду та зміни статусів замовлень.
  
  Інтеграція платіжних систем: Підключити API LiqPay та опцію банківського переказу.
  
  Система відгуків та обраного: Реалізувати можливість для користувачів залишати відгуки, ставити оцінки, додавати товари в обране та до порівняння.
  
  Мобільна адаптація: Забезпечити адаптивність веб-інтерфейсу для мобільних пристроїв.
