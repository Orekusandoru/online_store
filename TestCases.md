# Тест-кейси для Online Store

## Інструмент тестування: Postman

### Тест-кейс 1: Успішна реєстрація нового користувача

- **Назва:** Успішна реєстрація нового користувача.
- **Опис:** Перевірка, що новий користувач може успішно зареєструватися в системі через API.
- **Кроки:**
     **1.** Створити новий `POST` запит у Postman.
     **2.** Встановити URL: `http://localhost:5000/api/auth/register`.
     **3.** Перейти до вкладки "Body", вибрати `raw` та `JSON`.
     **4.** Додати тіло запиту з унікальним email та паролем:
     ```json
     {
         "name": "Test User",
         "email": "testuser12345@example.com",
         "password": "password123"
     }
     ```
  5. Надіслати запит.

- **Очікуваний результат:**
  - Статус-код відповіді: `201 Created`.
  - Тіло відповіді містить повідомлення про успішну реєстрацію та JWT токен.
  - Новий користувач створений у базі даних.

- **Фактичний результат:** Тест пройдено успішно. Отримано статус `201 Created`. У відповіді міститься JWT токен. Користувача успішно створено в базі даних.

### Тест-кейс 2: Отримання списку товарів

- **Назва:** Отримання списку всіх доступних товарів.
- **Опис:** Перевірка, що API повертає список товарів з каталогу.
- **Кроки:**
     **1.** Створити новий `GET` запит у Postman.
     **2.** Встановити URL: `http://localhost:5000/api/products`.
     **4.** Надіслати запит.

- **Очікуваний результат:**
  - Статус відповіді: `200 OK`.
  - Тіло відповіді є JSON-масивом, що містить об'єкти товарів.
  - Кожен об'єкт товару має поля `id`, `name`, `price` тощо.

- **Фактичний результат:** Тест пройдено успішно. Отримано статус `200 OK`. Тіло відповіді містить масив об'єктів товарів з усіма очікуваними полями.

---

### Тест-кейс 3: Спроба створення замовлення неавторизованим користувачем

- **Назва:** Спроба створення замовлення без авторизації.
- **Опис:** Перевірка, що система не дозволяє створювати замовлення неавторизованим користувачам.
- **Кроки:**
     **1.** Створити новий `POST` запит у Postman.
     **2.** Встановити URL: `http://localhost:5000/api/orders`.
     **3.** Перейти до вкладки "Body", вибрати `raw` та `JSON`.
     **4.** Додати тіло запиту з даними замовлення:
     ```json
     {
       "items": [{ "productId": 1, "quantity": 2 }],
       "customerInfo": { "name": "Guest", "phone": "123456789" },
       "deliveryInfo": { "address": "123 Main St" }
     }
     ```
  5. Переконатися, що у вкладці "Authorization" не встановлено жодного токена.
  6. Надіслати запит.

- **Очікуваний результат:**
  - Статус відповіді: `401 Unauthorized`.
  - Тіло відповіді містить повідомлення про помилку, наприклад, "Authentication token is missing or invalid".

- **Фактичний результат:** Тест пройдено успішно. Отримано статус `401 Unauthorized`. У тілі відповіді міститься повідомлення про помилку авторизації.
